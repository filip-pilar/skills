# Evaluation Case Title

## Purpose

State the behavior or regression this case protects.

## Target

Identify the skill version or accepted behavioral baseline.

## Invocation and context

Provide the realistic user request, inherited context, available tools, and relevant state. Keep suspected causes and desired fixes out of the execution prompt unless the case tests them directly.

## Raw evidence

Include or link the relevant input, output, artifact, diff, log, or trace. Distinguish observed evidence from interpretation.

## Expected behavior

Describe the observable successful outcome.

## Forbidden behavior

List material actions, assumptions, scope changes, or output properties that must not occur.

## Pass conditions

Write testable behavioral conditions. Use exact wording, size, or formatting only when it is genuinely part of the contract.

## Validation record

Record date, environment, verification-maturity level, result, and unresolved variance without placing the expected answer where a fresh evaluator can discover it.
