# Regression workspace

Keep only durable regression assets here: executable tests, minimal reusable scenarios with observable pass conditions, and fixtures those checks directly need.

For scenario runs, give the executor only the target and scenario input, then assess the hidden pass conditions. Keep transcripts and results ephemeral. Do not commit model outputs, judge reports, scores, hashes, dated run histories, or one-off comparisons.

Runtime-consumed examples and templates belong with the skill that loads them.
